# MCP-сервер

Подсистема «Сервер MCP» превращает базу 1С в набор инструментов для внешних
ИИ-агентов (Claude Desktop, Cursor, любой клиент [Model Context Protocol](https://modelcontextprotocol.io)).
Агент подключается по HTTP и работает с конфигурацией: читает метаданные,
выполняет read-only запросы, строит диаграммы.

## Состав

- HTTP-сервис `КИИ_СерверMCPAPI` с двумя точками:
  - `GET /health` — проверка живости (без авторизации не отдаёт ничего лишнего);
  - `POST /rpc` — JSON-RPC 2.0, методы `tools/list` и `tools/call`.
- Инструменты: `ping`, `get_configuration_info`, `get_metadata_tree`,
  `get_object_structure`, `execute_query` (только чтение), `generate_diagram`.
- Обработка «Сервер MCP: настройки» — пользователь API, пароль, проверка соединения.
- Роль `КИИ_СерверMCPДиагностика` — минимальные права для сервисного пользователя.
- Прокси [iikona-mcp-proxy](https://github.com/andromanpro) — мост stdio → HTTP
  для клиентов, которые не умеют HTTP-транспорт напрямую (Claude Desktop).

## Настройка

1. Опубликуйте базу на веб-сервере (стандартная публикация 1С, достаточно
   галки «HTTP-сервисы»).
2. Создайте сервисного пользователя ИБ и назначьте ему роль
   `КИИ_СерверMCPДиагностика` (плюс «Базовые права» ИИконы). Полные права
   не нужны — инструменты рассчитаны на least privilege.
3. В обработке «Сервер MCP: настройки» задайте логин/пароль API и проверьте
   соединение кнопкой.
4. Подключите агента: адрес `http(s)://<сервер>/<база>/hs/mcp/rpc`,
   Basic Auth с учётными данными из шага 3. Для Claude Desktop используйте
   прокси `iikona-mcp-proxy` (настройка в его README).

## Безопасность

⚠ **Basic Auth передаёт учётные данные в каждом запросе. Наружу сервис
выставляется только через HTTPS** (reverse-proxy с TLS — nginx/Apache/IIS
перед веб-сервером 1С). HTTP допустим только внутри доверенной сети.
Guard на стороне 1С это не проверяет — за транспортное шифрование отвечает
инфраструктура публикации.

Что уже делает сама подсистема:

- **Авторизация до диспетчеризации**: неавторизованный запрос не доходит
  до инструментов.
- **Ограничение частоты**: 60 вызовов в минуту на вызывающего; при превышении —
  ответ с интервалом повтора.
- **Только чтение**: `execute_query` пропускает исключительно запросы выборки
  (whitelist-валидация текста), число строк результата ограничено.
- **Сокрытие секретов**: значения, похожие на ключи/пароли, в ответах
  маскируются.
- **Аудит**: каждый вызов пишется в журнал регистрации и регистр
  «Сервер MCP — Аудит вызовов» (кто, метод, хеш параметров, результат,
  параметры с замаскированными секретами). Регламентное задание
  «Сервер MCP: очистка устаревших записей аудита» удаляет записи старше
  30 дней — при необходимости другого срока хранения отключите задание
  и чистите регистр своим регламентом.

## Ограничения

- Инструменты работают со своей базой: изменение данных и выполнение
  произвольного кода не предусмотрены by design.
- Метаданные отдаются по whitelist коллекций (справочники, документы,
  регистры, роли, перечисления, общие модули, обработки, отчёты).
